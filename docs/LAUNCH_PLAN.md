# WedScene AI 1.0 公测推进表

## 1.0 范围

1. 电影感分镜图：上传婚礼现场照，生成 6 张 16:9 分镜。
2. 类似婚礼：上传婚礼现场照，生成 6 张同风格参考图。
3. 只写标题文案：上传婚礼现场照，不生图，直接生成爆款标题、正文和话题。
4. 客户资源库：只展示图片和视频素材，支持预览、放大和逐张保存。
5. 客户页面：顶部菜单切换独立页面，保留产品、演示、资源库、公测说明和 FAQ。
6. 内部收费闭环：账号、点数、后台和支付规划只放在内部计划里，不展示给客户。

## 新功能计划

### 空场地融合婚礼图

目标：上传 1 张空场地照片 + 1 张婚礼参考图，把婚礼参考图的色系、花艺、灯光、舞台/通道关系融合到空场地里，输出真实可落地的婚礼现场候选图。

建议形态：

1. 输入：空场地图作为空间和结构主参考，婚礼图作为风格参考。
2. 输出：4 张 16:9 真实现场候选图，不做上下对比图。
3. 重点控制：保留空场地的墙面、层高、柱子、门窗、地面方向和舞台位置；迁移婚礼图的主色、花艺密度、灯光氛围、材质和仪式区设计语言。
4. 适用场景：客户看场后快速提案、酒店宴会厅方案预演、同一设计风格落到不同场地。
5. 命名备选：空场融合婚礼、空场地生成婚礼、场地方案融合、空场实景设计。
6. 技术要点：前端支持双图上传，后端保留两张参考图；提示词明确“空场地控制空间，婚礼图控制风格”，避免把参考婚礼的原场地直接复制过来。

## 公测运营建议

1. 先邀请 5-10 个真实婚礼客户或同行团队使用。
2. 先人工开通账号和点数，避免一开始就被支付、退款、对账拖慢。
3. 每天记录失败任务、客户满意标题、客户不满意图片类型。
4. 每周复盘一次成本：生图接口费用、服务器费用、失败率、付费意向。
5. 保留人工客服入口，公测期客户的问题比自动化更重要。

## 账号系统最小表结构

先不急着实现复杂会员，建议最小结构：

| 表 | 用途 |
|---|---|
| users | 用户账号、手机号/邮箱、状态 |
| credit_accounts | 用户点数余额 |
| credit_transactions | 充值、扣点、补偿流水 |
| jobs | 生成任务、状态、消耗点数、错误信息 |
| resources | 我的资源、标题、模式、文件列表 |
| orders | 后续支付订单、金额、状态、回调记录 |

## 上线前检查

1. `.env` 不进 GitHub。
2. API key 只放服务器环境变量。
3. `/api/health` 不返回密钥。
4. 服务器防火墙只开放 80、443、SSH。
5. 上传大小有限制，目前是 10MB。
6. 生成失败可以显示原因，并保留已完成图片。
7. 客户资源库只展示图片/视频素材，并支持逐张保存。
8. 内部点数、账号、收费闭环不展示给客户，只保留在运营文档里。
